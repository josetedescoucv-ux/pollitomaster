# Pollito Master 0.0.1 Release Notes

Release checkpoint for Pollito Master version 0.0.1.

## Desktop App Foundation

- React + TypeScript application running inside a Tauri v2 desktop shell.
- Desktop window configured for Pollito Master with a 1400x900 default size.
- Vite production output is generated in the project root `dist` directory.
- Tauri production packaging reads web assets from `../dist`.

## Tauri Integration

- Tauri v2 is configured as the desktop foundation.
- Application identifier is `com.josetedesco.pollitomaster`.
- Native SQL plugin is registered in the Tauri runtime.
- App icons are present for Windows, macOS, Linux, Android, and iOS bundle targets generated by the Tauri icon workflow.

## Installer Generation

- Production builds generate a release executable at `src-tauri/target/release/pollito-master.exe`.
- Windows MSI installer generation is configured and verified.
- Windows NSIS setup executable generation is configured and verified.
- Installer outputs are written under `src-tauri/target/release/bundle`.

## Current Implemented Features

- Auth flow with local email/password accounts and demo Google sign-in.
- Protected application routing.
- Home dashboard.
- Planner workspace with event management.
- Alarm center.
- AI assistant and provider connection management.
- Settings page with profile, theme, language, layout, alarm defaults, backup, and developer sections.
- Local storage persistence with a Tauri SQLite service path available in desktop runtime.
- JSON backup/export style persistence support.

## Known Limitations

- Authentication is local/demo-oriented and not a hardened production identity system.
- AI provider keys are stored locally by the current app implementation.
- SQLite integration is available for the Tauri runtime, while browser development can fall back to local storage.
- Desktop notifications and background alarm scheduling are not yet implemented as native background services.
- Large frontend bundle warning is present during Vite production builds; it does not block packaging.

## Verification

The 0.0.1 checkpoint should be verified with:

```bash
npm run build
npm run tauri build
```
